3.  [Docker Installation](https://docs.docker.com/engine/install/ubuntu/)
    First, update your package index and install prerequisites:

    ```bash
    sudo apt-get update
    sudo apt-get install -y ca-certificates curl software-properties-common
    ```

    Next, add Docker's official GPG key:

    ```bash
    sudo mkdir -p /etc/apt/keyrings
    curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
    ```

    Add the Docker repository to APT sources:

    ```bash
    echo \
    "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
    $(lsb_release -cs) stable" | \
    sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
    ```

    Update your package index again now that Docker's repo has been added:

    ```bash
    sudo apt-get update
    ```

    Finally, install Docker Engine, Docker CLI, and other necessary components:

    ```bash
    sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
    ```

4.  [Anaconda Installation](https://docs.anaconda.com/free/anaconda/install/linux/)
    Begin by installing the required libraries to ensure compatibility with your system:

    ```bash
    sudo apt-get update
    sudo apt-get install -y libgl1-mesa-glx libegl1-mesa libxrandr2 libxss1 libxcursor1 libxcomposite1 libasound2 libxi6 libxtst6
    ```

    Next, download the Anaconda installation script. Ensure you're placing the script in the `~/Downloads` directory correctly:

    ```bash
    curl -o ~/Downloads/Anaconda3-2024.02-1-Linux-x86_64.sh https://repo.anaconda.com/archive/Anaconda3-2024.02-1-Linux-x86_64.sh
    ```

    Change the permissions of the downloaded script to make it executable:

    ```bash
    chmod +x ~/Downloads/Anaconda3-2024.02-1-Linux-x86_64.sh
    ```

    Now, run the Anaconda installation script:

    ```bash
    bash ~/Downloads/Anaconda3-2024.02-1-Linux-x86_64.sh
    ```

    Follow the on-screen prompts to complete the installation. It's recommended to accept the prompt to initialize Anaconda3 by running `conda init`, as this will set up the shell to use conda directly.
    After installation, you may need to close and reopen your terminal or source your profile file (e.g., `source ~/.bashrc`) to activate the Anaconda environment.

### Setting Up and Running Your Project in a Conda Environment with Python 3.12

Creating and activating a dedicated Conda environment for your project ensures that all dependencies are isolated, making your development setup cleaner and more manageable. Follow these steps to set up your project environment:

1. **Creating and Activating Your Conda Environment:**
   If you do not have a Conda environment set up for this project, create one using the following command. This command creates an environment named `myenv` with Python 3.12. Feel free to replace `myenv` with your preferred environment name:

   ```bash
   conda create -n myenv python=3.12 -y
   conda activate myenv
   ```

   This command first creates the environment and then activates it. If you already have an environment, simply activate it with `conda activate your_env_name`.

2. **Installing Pipenv:**
   With your Conda environment activated, install Pipenv using pip. Pipenv is a dependency manager that creates a virtual environment for your Python projects, combining the best features of pip and virtualenv:

   ```bash
   python -m pip install pipenv
   ```

3. **Setting Up Your Project Environment with Pipenv:**
   In your project directory, initialize your environment with Pipenv to install dependencies as specified in your `Pipfile`. The `-v` flag provides verbose output, giving more details about the installation process:

   ```bash
   python -m pipenv install -v
   ```

4. **Activating the Pipenv Shell:**
   To switch to the virtual environment created for your project by Pipenv, activate the Pipenv shell:

   ```bash
   python -m pipenv shell
   ```

   This shell is specific to your project's environment and ensures that all Python commands and scripts are executed within this isolated environment.

5. **Running Your Application:**
   With the project's virtual environment activated, you can start your application. For example, to run a FastAPI application using `uvicorn` as the ASGI server, use:

   ```bash
   uvicorn your_project.server:app --port 3000
   ```

   Adjust `your_project.server:app` to the appropriate Python module path for your application.

6. **Setting Up the Frontend:**
   If your project includes a frontend that uses Node.js and npm, navigate to the frontend directory from a new terminal or tab (to keep the backend server running). Install the required npm packages:

   ```bash
   cd frontend
   npm install
   ```

7. **Running the Frontend Development Server:**
   Start the frontend development server to see your frontend application in action. This server typically supports live reloading, automatically reflecting changes you make to the code:

   ```bash
   npm start
   ```

   This command should be run outside of your Pipenv shell but can be within your Conda environment if it contains dependencies required by your frontend.


### Resuming Your Project Work

1. **Activate Your Conda Environment:**
   Whenever you start a new terminal session and want to work on your project, the first step is to activate the Conda environment you created for your project. Use the command below, replacing `myenv` with the name of your Conda environment:

   ```bash
   conda activate myenv
   ```

   This step ensures that any Python or command-line tools you use are limited to the dependencies and versions installed within this environment, maintaining project consistency and avoiding conflicts with other projects.

2. **Activate Your Pipenv Environment:**
   Next, navigate to your project directory where your `Pipfile` exists and activate the Pipenv environment. This step is crucial if you're working on Python projects managed by Pipenv, as it ensures you're using the correct versions of Python packages specified for your project:

   ```bash
   cd /path/to/your/project
   python -m pipenv shell
   ```

   This command activates the virtual environment for your project, configured by Pipenv. You'll need to do this every time you work on your project to ensure you're using the right Python environment.

3. **Start Your Backend Application:**
   If your project includes a backend component (like a FastAPI application), start it with the relevant command. For a FastAPI app using Uvicorn, for example:

   ```bash
   uvicorn your_project.server:app --port 3000
   ```

   Make sure to replace `your_project.server:app` with the path to your application's main module. This command needs to be run from within the Pipenv shell activated in step 2.

4. **Start Your Frontend Development Server:**
   If you have a frontend that runs separately (like a React or Vue application), you'll need to open a new terminal or a new tab. If the frontend requires any environment configurations from the Conda environment, activate the Conda environment again with `conda activate myenv`. Then navigate to your frontend directory:

   ```bash
   cd /path/to/your/frontend
   npm start
   ```

   This step launches the development server for your frontend, often with live reloading enabled, so you can continue working on the frontend with immediate feedback on changes.
